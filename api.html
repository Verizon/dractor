class dractor.wsman.HTTPConfig(connection_timeout=None, read_timeout=None, max_retries=None, verify_ssl_cert=None)¶

Configuration settings controlling the usage of HTTP by the WSMANClient

max_retries¶

int – The maximum number of retries allowed for failed HTTP requests.

timeouts¶

tuple – A tuple containing the connection_timout and read_timeout values.

Note

The max_retries field will indicate the maximum number of retries that should be made for ‘failed’ HTTP requests. In this context, an HTTP request is considered to be a ‘failed’ one, if the HTTP status code was not in the success range (e.g. 200s).
